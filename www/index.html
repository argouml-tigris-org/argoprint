ArgoPrint is a project of a course at Linköping Institute of Technology, Sweden. The initiative behind this project is Linus Tolke, one of the main developers of ArgoUML.

The aim of this project is to produce a module for ArgoUML that can automatically generate reports of the data contained in an ArgoUML project. The module will read data from a template that is written by the document author in an XML-format defined by the project. The template defines the content and format (for example DocBook) of the output document and makes it possible to extract information from the ArgoUML model to be included in the document.

The reason for doing this is that it will enable a completely automatic way of generating the documentation for a project modelled in ArgoUML and it will make it much easier to maintain the documentation even when changes are made to the ArgoUML model. All you need to do to keep the documentation up to date is to run the templates through ArgoPrint and the changes will be reflected in the documents.
